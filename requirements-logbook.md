# 日志功能 — 需求文档

## 1. 概述
在现有博客站点上新增一个「日志」页面，以日历形式展示博客文章的更新记录。数据自动从博客文章的 `pubDate` 字段生成，无需手动维护。

## 2. 导航入口
- 导航栏新增一个链接：「日志」，href="/log"
- 位置在「博客」和「关于」之间
- 代码位置：`src/consts.ts` 的 `NAV_LINKS` 数组

## 3. 页面路径
- `src/pages/log/index.astro`

## 4. 日历组件
### 4.1 布局
- 当前月份显示为一个 7列 × 最多6行 的日历网格
- 顶部显示当前月份名称和年份（如 "2026年6月"）
- 左右各一个箭头按钮切换月份
- 周日历头：日 一 二 三 四 五 六

### 4.2 格子规则
- 每个日期一个格子，显示日期数字
- 该日期有博客文章（匹配 `pubDate` 的年月日）→ 格子**填充绿色**，文字白色
- 无文章的日期 → 保持空白（浅灰背景或透明，参考现有设计风格）
- 非当前月份的空白天数 → 灰色不可用状态
- 当天日期 → 可额外加一个圆环或边框高亮（区别于"有文章"的绿色）

### 4.3 点击交互
- 点击绿格 → 无特殊动作（数据统一展示在下方列表）
- 点击空白格 → 无动作

## 5. 日志列表（日历下方）
### 5.1 展示逻辑
- 日历下方固定展示当前月份**所有**有文章的日子
- 按日期倒序排列（最新的在上面）
- 每一天作为一个条目组，组标题显示日期（如 "6月15日"）
- 该组下列出当天的所有文章

### 5.2 文章条目格式
- 文章标题（链接到文章页）
- 文章分类标签（如果有）

### 5.3 切换月份联动
- 切换到上个月／下个月时，日历和下方列表同步刷新
- 如果切换到的月份没有任何文章 → 日历显示正常（全空白格），下方列表显示 "该月暂无更新"

## 6. 技术方案
- 纯静态生成（无需 JS 运行时请求）
- 在 Astro 页面中通过 `getCollection("blog")` 获取所有文章
- 获取所有文章的 `pubDate`，按年月分组并标记有文章的日期
- 日历渲染：服务端生成的 HTML 网格
- 月份切换：通过 URL query parameter 实现（如 `/log?month=2026-06`），纯静态无 JS 切换
  - 或者客户端 JS 实现切换（更流畅），用 `<script>` 块处理
- 样式复用现有 Tailwind 紫色主题 + 深浅色模式

- 深色模式支持
- 简洁，保持博客现有设计语言
- 日历格子大小统一，响应式
